Семантический тег - это теги которые подскзаывают браузеру где какие элементы находятся. 
Это нужно для поисковика гугла, чтобы он лучше понимал сайт и тем лучше он будет ранжировать его в поиске.
Есть два вида списков в html, по крайней мере о которых я узнал. Это ul и ol теги.
Внутри этих тегов обязательно нужны теги li,
которые и будут как раз-таки самими элементами списков.
Они отличаются тем, что ul - это список маркерами, а ol - это список цифрами и буквами, включая римские цифры.

padding - это внутренний отступ контейнера
border - рамка контейнера
margin - внешний отступ за контейнером


У CSS есть элементы блочные и строчные.
Самый популярный строчный - это тег <a>
Дальше наверное тег <img>
Строчные элементы располагаются рядом то тех пор, пока есть место.
Блочные элементы занимают всё доступное место в ширину. Внутрь блочного элемента можно помещать как строчные элементы, так и блочные.
А внутрь строчных максимум строчные, блочные нельзя

У строчных и блочных может меняться поведение, когда меняем отступы

div {
    background-color: green;
    /* По часовой стрелке верх, право, низ, лево */
    margin: 30px 50px 50px 0px;
    padding: 20px 40px 0px 0px;
    /*Здесь размер рамки, потом вид рамки, цвет рамки*/
    border: 3px solid black;
    width: 500px;
    /*Так можно из блочного сделать строчныйblock
    display: inline;*/
    /*display: inline-block*/
}
.test {
    /*Это не работает, т.к. это свойства не для 
    строчного элемента */

**Схлопывание margin**
Схлопывание происходит, когда два маржина направленные друг к другу имеют допустим разный размер, один блок 100 пикселей вниз, другой блок ниже находящийся 40 пикселей вверх, в итоге между ними будет расстояние в 100 пикселей. Это и есть схлопывание маржинов.
Остановка 3.14

Маржины следует задавать сверху вниз и слева направо. Марджин топ и марджин лефт стараемся не использовать.
Правило 1: делай марджины справа и снизу (margin-right, margin-bottom)
Правило 2: Элемент не может двигать сам себя, его двигает сосед сверху или слева

**Псевдоклассы**
Можно избавиться от марджина последнего в списке li
псевнокласс hover - это для наводки. Когда наводимся на элемент.

Если нет соседнего элемент сверху или слева, мы используем padding top или padding left у "родителя"
Последнему элементу группы нужно обнулить отступ margin с помощью псевдоклассов (selector:last-child {margin: 0px;})

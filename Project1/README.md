Семантический тег - это теги которые подскзаывают браузеру где какие элементы находятся. Это нужно для поисковика гугла, чтобы он лучше понимал сайт и тем лучше он будет ранжировать его в поиске. Есть два вида списков в html, по крайней мере о которых я узнал. Это ul и ol теги. Внутри этих тегов обязательно нужны теги li, которые и будут как раз-таки самими элементами списков. Они отличаются тем, что ul - это список маркерами, а ol - это список цифрами и буквами, включая римские цифры.

padding - это внутренний отступ контейнера border - рамка контейнера margin - внешний отступ за контейнером

У CSS есть элементы блочные и строчные. Самый популярный строчный - это тег  Дальше наверное тег  Строчные элементы располагаются рядом то тех пор, пока есть место. Блочные элементы занимают всё доступное место в ширину. Внутрь блочного элемента можно помещать как строчные элементы, так и блочные. А внутрь строчных максимум строчные, блочные нельзя

У строчных и блочных может меняться поведение, когда меняем отступы

div \{ background-color: green; /\* По часовой стрелке верх, право, низ, лево \*/ margin: 30px 50px 50px 0px; padding: 20px 40px 0px 0px; /*Здесь размер рамки, потом вид рамки, цвет рамки*/ border: 3px solid black; width: 500px; /*Так можно из блочного сделать строчныйblock* *display: inline;*/ /*display: inline-block*/ \} .test \{ /\*Это не работает, т.к. это свойства не для строчного элемента \*/ margin: 50px 0px 50px 0px; width: 500px; \}

**Схлопывание margin** Схлопывание происходит, когда два маржина направленные друг к другу имеют допустим разный размер, один блок 100 пикселей вниз, другой блок ниже находящийся 40 пикселей вверх, в итоге между ними будет расстояние в 100 пикселей. Это и есть схлопывание маржинов. Остановка 3.14

Маржины следует задавать сверху вниз и слева направо. Марджин топ и марджин лефт стараемся не использовать. Правило 1: делай марджины справа и снизу (margin-right, margin-bottom) Правило 2: Элемент не может двигать сам себя, его двигает сосед сверху или слева

Поэтому нужно знать СВОЙСТВА background для изображений:

background-image — свойство для указания пути к фоновому изображению

background-position — свойство для указания месторасположения изображения в контейнере

background-size — свойство для указания размера фонового изображения

background-repeat — свойство которое обязывает повторяться изоброжение на все свободное место контейнера

background-attachment — свойство которое настраивает отображение при прокрутке страницы

<img> стоит использовать тег для изображений, которые являются частью контента (логотип, иконки на соц.сети, картинки примеров, картинки ссылки)

изображения свойством css background используем для изображений которые являются стилистическим оформлением сайта (картинка фона, картинка оформления)

Есть сокращение для background

background: url() top left/50px 50px no-repeat scroll
где все параметры: background-image, background-position, background-size, background-repeat, background-attachment.

Все свойства пишутся через пробел, кроме position и size, их необходимо разделять знаком слеш.
Порядок написания значений может быть и производильным, браузер сам определит соответствие свойств и значений.
